Python面向对象编程基础(五)

继承(Inheritance)

Posted by     "Toby" on Friday, June 12, 2020

一.什么是继承

生活中: 继承的含义一般是子女继承父辈的财产。

程序中: 继承(Inheritance)是指,允许一个类(子类或派生类)继承另一个类(父类或基类)的属性和方法。通过继承,可以实现代码的重用和扩展,提高了软件的可重用性和可维护性。

在面向对象编程中: 继承主要有三种类型:单继承(Single Inheritance)、多继承(Multiple Inheritance)和多级继承(Multilevel Inheritance)。

重要概念:

  • 父类(基类): 是已经定义好的类,它可以包含数据(属性)和行为(方法)
  • 子类(派生类): 是从父类继承而来的新类,它可以继承父类的属性和方法
  • 子类和父类的关系: 子类还可以添加自己的新属性和方法,或者重写(覆盖)继承自父类的方法。

简而言之，在程序中，继承描述的是多个类之间的所属关系，如果一个类A里面的属性和方法可以复用，则可以通过继承的方式，传递到类B里。那么类A就是基类(父类)；类B就是派生类(子类)

二.单继承(Single Inheritance)

是指一个子类只能继承一个父类。这种继承方式是最简单、最常见的继承形式。

结构:

class 子类名(父类名):
    # 子类的属性和方法

demo：

# 定义父类
class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        print("The animal speaks")

# 定义子类Dog,继承自Animal类
class Dog(Animal):
    pass

# 创建Dog对象并调用方法
dog = Dog("Buddy")
print(dog.name)  # 输出: Buddy
dog.speak()  # 输出: The animal speaks

虽然子类 Dog 没有定义 __init__ 方法初始化属性，也没有定义实例方法 speak ，但是可以直接可以从父类中继承过来。

三.多继承(Multiple Inheritance)

1.多继承作用和结构

是指一个子类可以同时继承多个父类的属性和方法。

结构:

class 子类名(父类1, 父类2, ...):
    # 子类的属性和方法

demo:

class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        print("The animal speaks")

class Flyable:
    def fly(self):
        print("I can fly")

class Bird(Animal, Flyable):
    pass

bird = Bird("Parrot")
print(bird.name)  # 输出: Parrot
bird.speak()  # 输出: The animal speaks
bird.fly()  # 输出: I can fly

2.多继承可能会引发下列的问题

2.1.菱形继承问题(Diamond Inheritance Problem)

当一个类继承自两个其他类，而这两个类又继承自同一个祖先类时，就形成了一个菱形结构。这可能导致属性和方法的重复继承,引发歧义和冲突。

解决方法: Python采用了深度优先的方法解析顺序(MRO)来解决这个问题,但仍需要格外小心。

demo:

class A:
    def method(self):
        print("A method")

class B(A):
    def method(self):
        print("B method")

class C(A):
    def method(self):
        print("C method")

class D(B, C):
    pass

d = D()
d.method()  # 输出结果取决于方法的解析顺序

2.2.命名冲突(Name Clashes)

当多个父类中有相同名称的属性或方法时，子类可能无法准确确定应该使用哪一个。这可能导致意外行为或错误。

解决方法: 在子类中必须显式重写这个方法,否则会引发歧义。

demo:

class A:
    def method(self):
        print("A method")

class B:
    def method(self):
        print("B method")

class C(A, B):
    pass

c = C()
c.method()  # 可能会引发冲突或者输出不符合预期的结果

2.3.显示调用父类构造函数

在 Python 中，如果一个子类继承自多个父类（多重继承），通常情况下子类的构造函数应该显式调用所有父类的构造函数，以确保父类的属性和方法得以正确初始化。这是因为每个父类可能有自己的初始化逻辑，如果不显式调用所有父类的构造函数，可能会导致部分父类的初始化代码未执行，从而导致意料之外的行为或错误。

在 Python中，可以使用 super() 函数来调用父类的构造函数。super() 函数会动态查找方法解析顺序(Method Resolution Order，MRO)，确保按照正确的顺序调用父类的构造函数。

demo:

class Parent1:
    def __init__(self):
        print("Parent1 constructor")

class Parent2:
    def __init__(self):
        print("Parent2 constructor")

class Child(Parent1, Parent2):
    def __init__(self):
        super().__init__()  # 显式调用第一个父类的构造函数
        Parent2.__init__(self)  # 显式调用第二个父类的构造函数
        print("Child constructor")

child = Child()

运行结果:

Parent1 constructor
Parent2 constructor
Child constructor

2.4.可读性和可维护性降低

多继承会增加代码的复杂性,可能导致可读性和可维护性降低。过度使用多继承可能使代码变得难以理解和修改。

2.5.Mixin 的使用不当

Mixin 是一种设计模式，用于在类之间共享方法和属性。但是，如果过度使用 Mixin 或者 Mixin 的设计不当，可能会导致类之间的耦合性增加，使代码更加脆弱和难以理解。

四.__mro__方法

作用: 方法是 Python 中用于获取类的方法解析顺序（Method Resolution Order，MRO）的特殊方法。MRO 定义了在多继承情况下，Python 解释器查找方法和属性的顺序。

demo:

class A:
    pass

class B(A):
    pass

class C(A):
    pass

class D(B, C):
    pass

print(D.__mro__)

运行结果:

(<class '__main__.D'>, <class '__main__.B'>, <class '__main__.C'>, <class '__main__.A'>, <class 'object'>)

在这个示例中，D 类的方法解析顺序依次为 D, B, C, A, object。这意味着在搜索方法时，Python 解释器会先在 D 类中查找，然后依次在 B, C, A 类中查找，最后在 object 类中查找。

注意:

如果多个父类中有同名的 属性和方法，则默认使用第一个父类的属性和方法（根据类的魔法属性mro的顺序来查找）

五.多级继承(Multilevel Inheritance)

是指一个子类继承自另一个子类,形成了一个继承链。在多级继承中,特性和行为沿着继承链向下传递。

结构:

class 基类:
    # 基类的属性和方法

class 派生类1(基类):
    # 派生类1的属性和方法

class 派生类2(派生类1):
    # 派生类2的属性和方法

demo:

class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        print("The animal speaks")

class Dog(Animal):
    def __init__(self, name, breed):
        super().__init__(name)
        self.breed = breed

    def bark(self):
        print("The dog barks")

class Puppy(Dog):
    def __init__(self, name, breed, age):
        super().__init__(name, breed)
        self.age = age

    def speak(self):
        print("The puppy barks cutely")

# 创建实例
puppy = Puppy("Buddy", "Labrador", 3)
puppy.speak()  # 输出: The puppy barks cutely
puppy.bark()  # 输出: The dog barks
print(puppy.name)  # 输出: Buddy
print(puppy.breed)  # 输出: Labrador
print(puppy.age)  # 输出: 3

在上面的示例中:Animal是基类,定义了name属性和speak()方法。Dog是派生类,继承自Animal类。它添加了breed属性和bark()方法,同时重写了__init__方法来初始化从基类继承的name属性。Puppy是另一个派生类,继承自Dog类。它添加了age属性,并重写了speak()方法。创建了一个Puppy实例,可以访问从Animal继承的name属性,从Dog继承的breed属性和bark()方法,以及自身的age属性和重写的speak()方法。

注意:

多级继承的优点是可以充分地代码重用,形成一个清晰的类层次结构。但是,如果继承链过长,代码可能会变得难以阅读和维护。此外,多级继承也可能引入类似于多重继承的一些潜在问题,如菱形继承问题等。

六.子类重写父类的同名方法和属性

场景: 允许子类在不改变父类行为的情况下重写父类的属性和方法，修改或者添加新的行为，提供了灵活性和可扩展性。

操作方法: 如果子类和父类的方法名和属性名相同，则默认使用子类的

demo:

class Parent:
    def __init__(self):
        self.value = "Parent's value"

    def say_hello(self):
        print("Hello from Parent")

class Child(Parent):
    def __init__(self):
        self.value = "Child's value"  # 重写父类的属性

    def say_hello(self):
        print("Hello from Child")  # 重写父类的方法

parent = Parent()
print(parent.value)  # 输出：Parent's value
parent.say_hello()  # 输出：Hello from Parent

child = Child()
print(child.value)  # 输出：Child's value，子类重写了父类的属性
child.say_hello()  # 输出：Hello from Child，子类重写了父类的方法

在这个示例中，Parent 类有一个 value 属性和一个 say_hello 方法。Child 类继承了 Parent 类，并且重写了父类的 value 属性和 say_hello 方法。当创建 Parent 和 Child 类的实例并调用它们的方法时，可以看到子类重写父类的属性和方法的效果。

七.子类调用父类同名属性和方法

场景: 当子类和父类拥有同名属性或方法时，如何在子类中调用父类的同名属性或方法

以下两种方法都可以在子类中调用父类的属性和方法

1.使用父类类名.父类方法(self)

好处:

  1. 这种方式直接指定了父类的类名，可以明确地调用父类的方法。
  2. 适用于单继承的情况，或者在多继承中明确知道要调用哪个父类的方法。
  3. 在多继承情况下，如果知道要调用的是哪个父类的方法，并且确定不会出现方法解析顺序（MRO）的问题，可以选择使用这种方式。

demo:

class Parent:
    def method(self):
        print("Parent method")

class Child(Parent):
    def method(self):
        Parent.method(self)  # 调用父类的方法
        print("Child method")

child = Child()
child.method()  # 输出：
                # Parent method
                # Child method

2.通过 super() 方法

好处:

  1. super() 是一个特殊的函数，用于动态地获取父类，可以自动处理方法解析顺序（MRO）。
  2. 在多重继承的情况下，super() 更加灵活，可以确保调用方法的正确版本，而不需要硬编码父类的类名。
  3. 适用于多继承的情况，特别是当父类的类名可能会发生变化时，使用 super() 更加安全。

在 Python 中，super() 可以在两种形式下使用：

2.1.无参数形式的 super()

demo:

class Parent:
    def method(self):
        print("Parent method")

class Child(Parent):
    def method(self):
        super().method()  # 调用父类的方法

child = Child()
child.method()  # 输出：Parent method

2.2.指定类和实例的 super()

demo:

class Parent1:
    def method(self):
        print("Parent1 method")

class Parent2:
    def method(self):
        print("Parent2 method")

class Child(Parent1, Parent2):
    def method(self):
        super(Parent1, self).method()  # 调用 Parent1 类的方法
        super(Parent2, self).method()  # 调用 Parent2 类的方法

child = Child()
child.method()  # 输出：
                # Parent1 method
                # Parent2 method

在这两个示例中，super() 方法被用来调用父类的方法，第一个示例中的 super() 使用了无参数形式，而第二个示例中的 super() 则指定了类和实例

总的来说，对于单继承的情况，直接使用父类类名来调用父类的方法可能更加清晰明了；而对于多继承的情况，尤其是在涉及到多个父类、可能会更改父类结构或者需要考虑方法解析顺序的情况下，推荐使用 super()。 super() 也有助于提高代码的可维护性和可扩展性。
