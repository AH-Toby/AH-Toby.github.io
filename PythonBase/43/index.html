Python面向对象编程基础(四)

封装(Encapsulation)

Posted by     "Toby" on Friday, June 12, 2020

一.面向对象的三大特性

封装(Encapsulation):

  • 将数据和代码捆绑在一个对象中,对外部隐藏对象的内部细节,仅暴露必要的接口供外部访问。
  • 使用类来定义对象,通过定义类的属性和方法来实现封装。
  • 还可以通过访问控制(如public、protected、private)来限制数据和方法的访问权限。

继承(Inheritance):

  • 一个类可以从另一个类继承其属性和方法,从而实现代码的重用。
  • 子类(派生类)可以继承父类(基类)的属性和方法,并可以添加自己的新属性和方法。
  • 多继承也是允许的,一个子类可以继承多个父类的属性和方法。

多态(Polymorphism):

  • 指同一操作作用于不同对象,可以有不同的解释和不同的执行结果。
  • 通过方法重写和方法重载来实现多态性。
  • 方法重写是子类重新定义父类的方法,而方法重载是在同一个类中定义多个同名但参数不同的方法。
  • Python没有方法重载,因为Python函数可以接受任意数量的参数。

二.什么是封装

封装（Encapsulation）是面向对象编程的一个重要特性，它指的是将数据和操作封装在一个单独的实体（类）中，同时对外提供接口来访问和操作这些数据和操作。

作用: 将数据和相关的方法组合成一个独立的单元，通过接口来控制对数据的访问。这样可以隐藏数据的具体实现细节，提供一种保护机制，防止外部直接访问和修改对象的内部数据。

三.实现封装的主要方法

1.使用类和对象

这是最基本的封装方式,将数据和方法组合在一个类中,通过创建该类的对象来访问其中的数据和方法。例如:

demo:

class Person:
    def __init__(self, name, age):
        self.name = name  # 公有属性
        self._age = age   # 受保护属性,前缀单下划线

    def display(self):
        print(f"Name: {self.name}, Age: {self._age}")

p = Person("Alice", 25)
p.display()  # 访问对象的方法
print(p.name)  # 访问公有属性
# print(p._age)  # 受保护属性,不建议直接访问

2.使用私有属性和方法

所谓的私有属性和方法是指类特有属性和方法，外界对象不能访问。Python中没有像C++和Java中public 和 private 这些关键字来区别公有属性和私有属性。Python是以属性命名方式来区分，如果在属性和方法名前面加了2个下划线 __，则表明该属性和方法是私有权限，否则为公有权限。

结构:

__属性名/方法名

demo:

class Person:
    def __init__(self, name, age):
        self.name = name
        self.__age = age  # 私有属性

    def __display(self):  # 私有方法
        print(f"Name: {self.name}, Age: {self.__age}")

    def show(self):
        self.__display()  # 内部调用私有方法

p = Person("Alice", 25)
p.show()
# print(p.__age)  # 错误,无法直接访问私有属性
# p.__display()  # 错误,无法直接访问私有方法

注意:

类的私有属性 和 私有方法，都不能通过对象直接访问，但是可以在本类内部访问；

类的私有属性 和 私有方法，都不会被子类继承，子类也无法访问；

私有属性 和 私有方法 往往用来处理类的内部事情，不通过对象处理，起到安全作用。

3.修改私有属性的值

在Python中,直接修改对象的私有属性是不被允许的,因为私有属性是对外部隐藏的。但是,仍然有方法可以间接修改私有属性的值。

3.1.通过公有方法修改

最常见的方式是通过类中定义的公有方法来修改私有属性的值。这种方式可以在方法中添加任何必要的逻辑或数据验证。

demo:

class Person:
    def __init__(self, name, age):
        self.name = name
        self.__age = age

    def get_age(self):
        return self.__age

    def set_age(self, new_age):
        if new_age >= 0:
            self.__age = new_age
        else:
            print("Invalid age!")

person = Person("Alice", 25)
person.set_age(30)
print(person.get_age())  # 输出: 30

上面的例子就是通过公共方法get_age和set_age来修改和获取私有属性的值

3.2.通过属性(property)修改

使用@property装饰器定义getter和setter方法，可以像访问普通属性一样修改私有属性的值。

demo:

class Person:
    def __init__(self, name, age):
        self.name = name
        self.__age = age

    @property
    def age(self):
        return self.__age

    @age.setter
    def age(self, new_age):
        if new_age >= 0:
            self.__age = new_age
        else:
            print("Invalid age!")

person = Person("Alice", 25)
person.age = 30  # 相当于:person.age(30)
print(person.age)  # 输出: 30 # 相当于:print(person.age)

3.3.通过名称重整(Name Mangling)修改

伪私有属性和私有方法: Python在编译时会自动重命名带有双下划线前缀的私有属性,以避免与子类中的同名属性发生冲突。通过这种机制,我们可以直接访问和修改私有属性(不推荐这种做法,因为破坏了封装)。这种方式就导致了Python的私有方法和属性不是真正意义上的私有而是伪私有的。

结构:

对象名._类名__私有名称/方法名称

demo:

class Person:
    def __init__(self, name, age):
        self.name = name
        self.__age = age

person = Person("Alice", 25)
print(person._Person__age)  # 输出: 25
person._Person__age = 30
print(person._Person__age)  # 输出: 30

注意:

直接修改私有属性的值容易破坏封装,应该尽量避免这种做法。推荐的方式是通过公有方法或属性(property)来间接修改私有属性的值,这样可以在修改时添加必要的逻辑和数据验证。
