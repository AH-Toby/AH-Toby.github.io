Python面向对象编程基础(七)

实例和类的属性方法

Posted by     "Toby" on Sunday, June 14, 2020

一.实例对象和类对象

1.实例对象(Instance Object)

概念: 通过类创建的对象

特点:

  • 实例对象是通过类对象实例化得到的对象，也称为类的实例。
  • 每个实例对象拥有自己的实例属性和方法，可以独立地进行操作。
  • 实例对象通常在程序运行时根据类对象进行创建，每次创建都会产生一个新的实例对象。

2.类对象

在Python中，一切皆为对象，包括类也是对象。当我们定义一个类时，实际上是在内存中创建了一个类对象，这个类对象拥有自己的一些特性，例如类属性、方法等。

概念: 用于创建类的对象，定义类的模板

特点:

  • 类对象是在 Python 中用于创建类的对象，也称为类的模板。
  • 类对象包含了类的属性和方法的定义，可以被实例化为实例对象。
  • 类对象通常在程序运行时被创建，并且可以被多个实例对象共享。

二.实例属性和类属性

1.实例属性(Instance Attributes)

概念: 通过类创建的对象 又称为 实例对象, 实例对象的属性称为实例属性，记录对象各自的数据，不同对象的同名实例属性，记录的数据可能各不相同

特点:

  • 实例属性是属于特定实例的属性，只能通过实例来访问和修改。
  • 每个实例可以拥有不同的实例属性值，彼此之间互不影响。
  • 实例属性通常在实例化对象时通过构造函数 __init__() 中初始化

demo:

class MyClass:
    def __init__(self, instance_attribute):
        self.instance_attribute = instance_attribute

# 创建实例并初始化实例属性
obj1 = MyClass("Instance attribute of obj1")
obj2 = MyClass("Instance attribute of obj2")

# 访问实例属性
print(obj1.instance_attribute)  # 输出：Instance attribute of obj1
print(obj2.instance_attribute)  # 输出：Instance attribute of obj2

# 修改实例属性
obj1.instance_attribute = "Modified instance attribute of obj1"
print(obj1.instance_attribute)  # 输出：Modified instance attribute of obj1

2.类属性(Class Attributes)

概念: 类对象 所拥有的属性被称为 类属性可以使用 类对象 或 实例对象 访问 它被该类的所有实例对象所共有

特点:

  • 类属性是定义在类中的属性，被所有该类的实例共享。
  • 类属性通常在类的定义中使用，但也可以在类的方法中通过类名访问和修改。
  • 当类的任何一个实例修改了类属性的值，其他实例的该属性也会受到影响。
  • 类属性通常用于描述类的特性或者保存所有实例共享的状态信息。

demo:

class MyClass:
    class_attribute = "I am a class attribute"

# 访问类属性
print(MyClass.class_attribute)  # 输出：I am a class attribute

# 修改类属性
MyClass.class_attribute = "Modified class attribute"
print(MyClass.class_attribute)  # 输出：Modified class attribute

注意:

实例属性要求每个对象为其单独开辟一份内存空间来记录数据，而类属性为全类所共有仅占用一份内存，更加节省内存空间。

三.类属性使用注意

1.尽量避免类属性和对象属性同名

注意: 如果有同名对象属性，实例对象会优先访问对象属性

demo:

class MyClass:
    attribute = "Class Attribute"

    def __init__(self, attribute):
        self.attribute = attribute

# 创建实例对象
obj = MyClass("Instance Attribute")

# 访问对象属性
print(obj.attribute)  # 输出：Instance Attribute

# 访问类属性
print(MyClass.attribute)  # 输出：Class Attribute

2.类属性只能通过类对象修改

注意: 不能通过实例对象修改，以确保修改的是类属性而不是实例属性。

demo:

class MyClass:
    class_attribute = "Class Attribute"

# 通过类对象修改类属性
MyClass.class_attribute = "Modified Class Attribute"

# 通过实例对象访问类属性
obj = MyClass()
print(obj.class_attribute)  # 输出：Modified Class Attribute

3.类属性不能直接设置为私有

注意: 类属性通常用于描述类的特性或者保存所有实例共享的状态信息，并且可以被所有该类的实例共享。因此，将类属性设置为私有属性通常没有意义，因为私有属性主要用于限制对实例属性的访问。

虽然在 Python 中不能直接将类属性设置为私有属性，但是可以通过一些命名约定来模拟私有类属性，例如在属性名称前面添加一个下划线 _ 。这样的命名约定会告诉其他开发者该属性应该被视为私有属性，不应该直接访问。

demo:

class MyClass:
    _private_class_attribute = "Private Class Attribute"

四.实例方法、类方法和静态方法

1.实例方法(Instance Method)

概念: 实例对象的方法称为实例方法

特点:

  • 实例方法是定义在类中的方法，没有任何装饰器修饰。
  • 实例方法的第一个参数是 self，代表实例对象本身。
  • 实例方法可以访问和修改实例属性，也可以访问类属性。
  • 实例方法通常用于操作实例属性，以及对实例对象进行操作和处理。

demo:

class MyClass:
    def __init__(self, instance_attribute):
        self.instance_attribute = instance_attribute

    def instance_method(self):
        return self.instance_attribute

# 创建实例对象
obj = MyClass("Instance Attribute")

# 调用实例方法
print(obj.instance_method())  # 输出：Instance Attribute

2.类方法(Class Method)

概念: 定义在类中的一种特殊方法，它与类相关联，而不是与类的实例对象相关联。类方法可以访问和修改类属性，但不能直接访问实例属性。它通常用于操作类本身和类属性，执行类级别的操作或者初始化操作。

特点:

  • 装饰器: 类方法使用 @classmethod 装饰器来标识。
  • 第一个参数: 类方法的第一个参数是 cls，表示类本身，而不是实例对象。
  • 访问类属性: 类方法可以访问和修改类属性，因为它有一个指向类的参数。
  • 无需实例化: 类方法可以直接通过类名来调用，而不需要实例化类对象。
  • 初始化操作: 类方法通常用于执行类级别的初始化操作，或者在创建实例之前执行一些特殊的操作。

结构:

class MyClass:
    class_attribute = "Class Attribute"

    @classmethod
    def class_method(cls, parameter1, parameter2):
        # 执行类方法的代码
        pass

demo:

class Circle:
    pi = 3.14159

    def __init__(self, radius):
        self.radius = radius

    @classmethod
    def from_diameter(cls, diameter):
        radius = diameter / 2
        return cls(radius)

    def area(self):
        return self.pi * self.radius ** 2

# 创建 Circle 对象，通过直径创建
circle1 = Circle.from_diameter(10)
print(circle1.area())  # 输出：78.53975

在这个示例中，我们定义了一个 Circle 类，它有一个类方法 from_diameter()，用于通过直径创建圆。在方法内部，我们计算出半径并返回一个新的 Circle 对象。通过类方法，我们可以方便地创建对象并执行其他类级别的操作，而不需要实例化类对象。

3.静态方法(Static Method)

概念: 定义在类中的一种特殊方法，它与类和类的实例对象都无关，可以独立地被调用。静态方法通常用于实现与类相关的功能，但不需要访问类属性或实例属性。

特点:

  • 装饰器: 静态方法使用 @staticmethod 装饰器来标识。
  • 无需实例化: 静态方法可以直接通过类名来调用，而不需要实例化类对象。
  • 无需类参数: 静态方法不需要额外的参数，不需要 self 或 cls 参数。
  • 访问限制: 静态方法不能直接访问类属性或实例属性，因为它没有指向类或实例的参数。

结构:

class MyClass:
    @staticmethod
    def static_method(parameter1, parameter2):
        # 执行静态方法的代码
        pass

demo:

class MathUtility:
    @staticmethod
    def add(x, y):
        return x + y

    @staticmethod
    def subtract(x, y):
        return x - y

# 调用静态方法
print(MathUtility.add(5, 3))       # 输出：8
print(MathUtility.subtract(5, 3))  # 输出：2

在这个示例中，我们定义了一个 MathUtility 类，它包含了两个静态方法 add() 和 subtract()，用于执行加法和减法运算。这些静态方法可以直接通过类名来调用，而不需要实例化类对象，因为它们与类和实例对象无关。

五.实例方法,类方法和静态方法使用注意

1.选择合适的方法类型

根据方法需要操作的属性和行为，选择合适的方法类型

  • 实例方法：方法内部需要访问实例属性（实例方法内部可以使用类名.访问类属性）
  • 类方法：方法内部只需要访问类属性
  • 静态方法：方法内部不需要访问实例属性和内属性
  • 如果方法内部、既需要实例属性又要访问类属性应该使用实例方法，因为类只有一个，在实例方法内部可以使用类名.访问类属性

2.避免混淆

在类中定义同名的实例方法、类方法和静态方法时，要注意调用时会 优先执行最后定义的方法，避免混淆和意外行为。

demo:

class MyClass:
    def method(self):
        print("Instance Method")

    @classmethod
    def method(cls):
        print("Class Method")

    @staticmethod
    def method():
        print("Static Method")

# 创建对象实例
obj = MyClass()

# 调用方法
obj.method()  # 输出：Static Method

应该修改为:

class MyClass:
    def instance_method(self):
        print("Instance Method")

    @classmethod
    def class_method(cls):
        print("Class Method")

    @staticmethod
    def static_method():
        print("Static Method")

# 创建对象实例
obj = MyClass()

# 调用方法
obj.instance_method()  # 输出：Instance Method
MyClass.class_method()  # 输出：Class Method
MyClass.static_method()  # 输出：Static Method
