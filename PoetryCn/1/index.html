Poetry简介

Posted by     "Toby" on Tuesday, July 4, 2023

Poetry是Python中用于管理依赖和打包的工具，支持您声明项目所需的库并负责管理其安装和更新。它提供锁定文件用于构建版本和发布项目以确保安装的一致性。

一.系统要求

Poetry支持Linux、macOS、Window等系统，但要求必须使用Python 3.8及以上版本。

二.安装

注意:

建议将Poetry 安装在一个专用的虚拟环境中，使其与系统的其余部分隔离开来。

绝不能将其安装在由 Poetry 管理的项目环境中。这可以确保 Poetry 的依赖项不会被意外升级或卸载。

1.使用pipx安装

pipx是一个用于管理 Python CLI 应用程序的工具，它能够在全局安装这些应用程序，并将它们隔离在独立的虚拟环境中。当使用pipx安装Poetry时，pipx将负责管理Poetry的升级和卸载。

1.1.安装pipx

如果尚未安装pipx，您可以按照官方pipx安装说明中的内容进行安装。任何非过时版本的pipx都可以。

1.2.安装Poetry

pipx install poetry

1.3.安装Poetry(高级安装)

pipx可以使用与pip相同的语法安装不同版本的Poetry

pipx install poetry==1.2.0

pipx还能同时安装不同版本的Poetry，这样就能轻松测试备用版本或预发布版本。安装时必须由用户指定一个 唯一的后缀，用于命名Poetry执行文件的名称如：

# 下载1.2.0版本并安装以@1.2.0作为后缀的poetry环境中
pipx install --suffix=@1.2.0 poetry==1.2.0  
poetry@1.2.0 --version

# 下载preview 版本并安装以@preview作为后缀的poetry环境中
pipx install --suffix=@preview --pip-args=--pre poetry
poetry@preview --version

pipx 还可以根据pip要求规范，从Git上下载安装开发版本，甚至在本地测试git拉取请求:

pipx install --suffix @master git+https://github.com/python-poetry/poetry.git@master
pipx install --suffix @pr1234 git+https://github.com/python-poetry/poetry.git@refs/pull/1234/head

1.4.更新Poetry

pipx upgrade poetry

1.5.卸载Poetry

pipx uninstall poetry

2.使用官方脚本安装

我们提供了一个自定义安装脚本，它创建一个新的虚拟环境，然后在其中安装Poetry，并允许Poetry管理自己的环境。

2.1.安装Poetry

安装的脚本可直接从install.python-poetry.org获取，并在自己的资源库中开发。该脚本可直接执行（例如"curl python"）或下载后从磁盘执行（如在 CI 环境中）。

注意:

原先的install-poetry.py 安装程序已被弃用移除。请使用上述版本进行安装。

Linux, macOS, Windows (WSL):

curl -sSL https://install.python-poetry.org | python3 -

注意：

在某些系统中，“python” 可能会指向 Python 2.x 版本的解释器，而"python3" 则明确表示使用 Python 3.x 版本的解释器。因此，使用 “python3” 命令可以确保你在执行 Python 3.x 版本而不是 Python 2.x 版本的命令。

Windows (Powershell):

(Invoke-WebRequest -Uri https://install.python-poetry.org -UseBasicParsing).Content | py -

注意：

如果通过 Microsoft Store 安装了 Python，请在上述命令中将 py 替换为 python。

2.2.安装Poetry(高级安装)

默认情况下，Poetry会安装到了特定的用户目录：

系统 安装目录
MacOS ~/Library/Application Support/pypoetry
Linux/Unix ~/.local/share/pypoetry
Windows %APPDATA%\pypoetry

如果你希望更改安装目录，你可以定义 $POETRY_HOME 环境变量：

curl -sSL https://install.python-poetry.org | POETRY_HOME=/etc/poetry python3 -

如果你想安装预发布版本，可以通过在安装脚本中传递 --preview 选项或使用 $POETRY_PREVIEW 环境变量来实现：

curl -sSL https://install.python-poetry.org | python3 - --preview
curl -sSL https://install.python-poetry.org | POETRY_PREVIEW=1 python3 -

同样，如果要安装特定版本，可以使用 --version 选项或 $POETRY_VERSION 环境变量：

curl -sSL https://install.python-poetry.org | python3 - --version 1.2.0
curl -sSL https://install.python-poetry.org | POETRY_VERSION=1.2.0 python3 -

你也可以使用 --git 选项从 git 仓库安装 Poetry：

curl -sSL https://install.python-poetry.org | python3 - --git https://github.com/python-poetry/poetry.git@master

2.3.添加Poetry环境变量

安装程序会在以下相关的目录中创建一个名为 poetry 的启动脚本：

系统 目录
Unix $HOME/.local/bin
Windows %APPDATA%\Python\Scripts
设置了$POETRY_HOME $POETRY_HOME/bin

如果以上目录不在你的 $PATH 中，你可以将它添加进去，这样就可以使用 poetry 命令来调用 Poetry。

你也可以直接使用poetry可执行文件完整的路径来调用如下：

系统 目录
MacOS ~/Library/Application Support/pypoetry/venv/bin/poetry
Linux/Unix ~/.local/share/pypoetry/venv/bin/poetry
Windows %APPDATA%\pypoetry\venv\Scripts\poetry
设置了$POETRY_HOME $POETRY_HOME/venv/bin/poetry

2.4.验证Poetry

一旦 Poetry 安装并加入了你的 $PATH，你可以执行以下操作：

poetry --version

如果你看到类似 Poetry (version 1.2.0)，说明你的安装已经完成

2.5.更新Poetry

如果你使用的是官方安装程序安装的 Poetry，你可以使用的它的自动更新命令

注意：

使用 self update 可能会有问题(特别是Windows系统)，因此最好使用安装程序进行重新安装。

poetry self update

如果你想更新到预发布版本，可以使用 --preview 选项。

poetry self update --preview

如果你想更新到特定版本，可以将其作为参数传递给 self update

poetry self update 1.2.0

注意：

Poetry 1.1 系列的版本无法直接升级到 1.2 或更新的系列版本。要迁移到更新的版本，需要使用下面的方法进行卸载，然后再使用上述方法进行重新安装。

2.6.卸载Poetry

如果你决定不再使用 Poetry，你可以通过再次运行安装程序并加上 --uninstall 选项，或者在执行安装程序之前设置 POETRY_UNINSTALL 环境变量从系统中完全的删除它。

curl -sSL https://install.python-poetry.org | python3 - --uninstall
curl -sSL https://install.python-poetry.org | POETRY_UNINSTALL=1 python3 -

注意：

如果你使用了已弃用的 get-poetry.py 脚本进行安装，你需要手动删除

rm -rf "${POETRY_HOME:-~/.poetry}"

同时，如果你配置了环境变量你还需要在shell 配置中删除$PATH中的 ~/.poetry/bin

3.手动安装(高级)

Poetry可以通过使用 pip 和 venv 模块手动安装。这样做实质上是执行了官方安装程序执行的步骤。由于这是一种高级安装方法，仅适用于Unix系统，这边就只介绍Poetry相关安装，系统相关准备不在这边赘述。

将使用变量 $VENV_PATH 来指示创建虚拟环境的路径。

python3 -m venv $VENV_PATH
$VENV_PATH/bin/pip install -U pip setuptools
$VENV_PATH/bin/pip install poetry

Poetry将被安装在 $VENV_PATH/bin/poetry 中，并且仅对 $VENV_PATH 生效，您可以直接调用该路径下的Poetry命令，或者在其他位置创建符号链接。

要卸载Poetry，只需删除整个$VENV_PATH目录即可。

4.CI环境安装建议

与开发环境通常倾向于使用最新的工具不同，CI环境可复现性才是最重要的，以下是一些建议，帮助你在 CI 环境中安装 Poetry。

4.1.版本固定

无论你使用哪种方法安装 Poetry，都建议您明确指定 Poetry 的版本，这样你就可以在自己的验证后进行升级。每种安装方法都有不同的语法来设置版本。 以下示例中将演示每种方法的版本控制。

pipx安装固定版本:

在CI环境中使用 Poetry，强烈推荐用 pipx进行安装。 pipx 非常给力，在 CI 环境中，绝对是使用 Poetry 的不二之选。

pipx install poetry==1.2.0

官方脚本安装固定版本:

注意：

官方安装脚本（install.python-poetry.org）提供了一种简化的 Poetry 安装方式，适用于开发者或简单的流水线。然而，在 CI 环境中，应该认真考虑另外两种支持的安装方法（pipx 和手动安装）。

建议在你的 CI 流水线能够访问的地方下载安装程序脚本的副本（或者保留存储库的副本），这样可以确保你的流水线稳定运行并对执行的代码保持控制。

默认情况下，安装程序会被安装到用户特定的目录。在更复杂的 CI 流水线中，这可能会导致访问 Poetry 变得有些困难，尤其是在像多阶段容器构建这样的情况下。在使用官方安装程序进行 CI 时，强烈建议使用 $POETRY_HOME，这样可以更精准地控制路径。

export POETRY_HOME=/opt/poetry
python3 install-poetry.py --version 1.2.0
$POETRY_HOME/bin/poetry --version

手动安装固定版本:

为了在你的CI环境中获得最大的控制权，强烈推荐使用pip进行安装。虽然这需要你提供更明确的命令和对Python打包的了解，但相应地提供了最佳的调试体验，并且减少了对外部工具的依赖。

export POETRY_HOME=/opt/poetry
python3 -m venv $POETRY_HOME
$POETRY_HOME/bin/pip install poetry==1.2.0
$POETRY_HOME/bin/poetry --version

注意:

如果你通过pip安装Poetry，请确保将Poetry安装在一个独立的环境中，而不是与Poetry管理的目标环境相同。 如果Poetry和你的项目安装在同一个环境中，Poetry可能会升级或卸载它自己的依赖项，这可能导致出现难以调试或无法预料的错误。

三.启动Tab自动补全(Bash,Fish,Zsh)

Poetry支持生成Bash、Fish和Zsh的补全脚本。

1.Bash

1.1.自动加载(推荐)

poetry completions bash >> ~/.bash_completion

1.2.懒加载

poetry completions bash > ${XDG_DATA_HOME:-~/.local/share}/bash-completion/completions/poetry

2.Fish

poetry completions fish > ~/.config/fish/completions/poetry.fish

3.Zsh

poetry completions zsh > ~/.zfunc/_poetry

注意：

如果 ~/.zshrc 文件中不存在以下两行代码，则需要在~/.zshrc文件中插入

fpath+=~/.zfunc
autoload -Uz compinit && compinit

4.Oh My Zsh

mkdir $ZSH_CUSTOM/plugins/poetry
poetry completions zsh > $ZSH_CUSTOM/plugins/poetry/_poetry

在~/.zshrc中的插件数组中添加poetry：

plugins(
    poetry
    ...
)

5.prezto

poetry completions zsh > ~/.zprezto/modules/completion/external/src/_poetry

注意：

为了使这些更改生效，您需要重新启动Shell。
